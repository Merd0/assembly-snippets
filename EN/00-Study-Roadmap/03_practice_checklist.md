# 03 - Practice Checklist (Per Session / Per Stage)

## Per session checklist

- [ ] I can explain the goal of today's file in one sentence.
- [ ] I know why each non-trivial instruction exists.
- [ ] I changed at least one value/branch/loop condition and observed effects.
- [ ] I wrote down one bug and how I fixed it.
- [ ] I can summarize today's key lesson in 3 bullets.

## Per stage checklist

- [ ] I ran every `.asm` file in the stage.
- [ ] I rewrote at least one file from memory.
- [ ] I added one small feature to one file.
- [ ] I reviewed register/stack/memory behavior with debugger/step mode.
- [ ] I can explain this stage to another beginner.

## Red flags (slow down if true)

- [ ] I copy code without understanding each instruction.
- [ ] I run once and move on without modifications.
- [ ] I skip stack/register analysis because output “looks right”.
- [ ] I cannot explain why a branch was taken.

If two or more red flags are true, repeat the stage before moving forward.
