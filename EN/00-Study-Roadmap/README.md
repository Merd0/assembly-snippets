# 00-Study-Roadmap

This guide answers one practical question:

> **"How should I actually study this repository from zero (or near-zero) to job-ready confidence?"**

It is designed to be **detailed but simple**: clear order, weekly rhythm, realistic workload, and concrete output expectations.

## Who is this for?

- You are new to low-level programming, or
- You know basic programming but are new to assembly, or
- You started assembly before but want a structured reboot.

## How to use this folder

1. Read `01_learning_path.md` once and choose your starting level.
2. Follow `02_weekly_plan.md` as your working schedule.
3. Use `03_practice_checklist.md` at the end of each study session.

## Expected outcome

If you follow this roadmap consistently, you should be able to:

- Explain what each register/memory operation does.
- Build and debug MIPS32 programs in MARS/QtSPIM.
- Read unfamiliar `.asm` code and reason about control/data flow.
- Implement mini-project algorithms with clean assembly style.
